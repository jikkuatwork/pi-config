- Complexity-only review (`/ponytail-review`, "review for over-engineering", "what can we delete", "is this over-engineered", "simplify review"): read `02_review_mode.md`.

## Boundaries

- Do not use Ponytail for broad architecture exploration, teaching, or alternatives unless the user asks for minimalism.
- Do not replace normal correctness/security/accessibility review; review mode hunts complexity only.
- Stop when user says "stop ponytail" or "normal mode".
- If user insists on the full/enterprise version, build it without re-arguing.
